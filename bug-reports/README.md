## 🧾 My Bug Report Standard

Every bug report I write — whether professional or sample — includes the following:

- **Clear one-line summary** that describes the issue precisely
- **Severity rating** — Critical, Major, Minor, or Trivial
- **Full environment details** — device, OS, browser/app version
- **Numbered reproduction steps** that anyone can follow
- **Expected vs. Actual result** — clearly separated
- **Screenshot or screen recording** attached as evidence
- **Additional notes** for intermittent issues or related observations

This format ensures developers can reproduce, understand, and fix the issue without needing to follow up with me for more information.
